# 訪問活動とチーム

## 訪問活動チーム

- 1人以上のメンバーでチームを組成できる。
  - 活動スタッフ・編集スタッフの両方がチームを組成できる（上位互換）
  - 1回の訪問活動ごとに組成できるし、再利用もできる。
  - チームはグループ横断で組成できる
  - 1人のメンバーが複数のチームに所属できる
- チーム責任者
  - 1人のときはその人が自動的に責任者
  - 2人以上のときは責任者を決める
    - 責任者は適宜変更可能
    - 決めないときは暗黙的に（通常はリスト要素の最初の人）が責任者になる。

## 訪問活動データ

- 訪問活動データ＝1つの区域の貸し出し（チェックアウト）
  - 訪問活動データが区域データを参照する
  - 訪問活動は1つの区域の網羅開始から終了まで1つのデータが紐づく
  - 1つの訪問活動に対して日時ごとに複数の訪問活動チームを紐づけることができる

### 区域の貸し出し（チェックアウト）モデル

- チェックアウトには2つの経路がある
  - **貸し出し**（編集スタッフ主体）: 編集スタッフが活動スタッフまたはチームに区域を貸し出す
    - 同じ区域を複数のチームに貸し出すことができる
  - **持ち出し**（活動スタッフ主体）: 活動スタッフが「持ち出し可能」区域を自主的に取得する
- 区域の可用性
  - 編集スタッフが網羅予定の中で、スコープに対して区域の可用性を設定する
  - **スコープ**（対象範囲）
    - グループ全員（スコープNULL）: LinkSelfグループ内の全メンバーが対象
    - 特定のグループ: 指定したグループのメンバーのみが対象
  - スコープに対して以下を設定する
    - **貸し出し可能区域**: ある期間に使用する区域を区域単位で設定する
    - **持ち出し可能区域**: 貸し出し可能区域の中から、活動スタッフが自主的に取得できる区域を設定する
      - 持ち出し可能 ⊂ 貸し出し可能（持ち出し可能は貸し出し可能の部分集合）
      - 貸し出し可能だが持ち出し不可の区域は、編集スタッフからの貸し出しでのみチェックアウトされる

### ステータス管理

- 開始前、活動中、完了済み
- 活動中に返却できる（未完了の状態で返却・回収）

### 返却と回収

- 返却（活動スタッフ主体）
  - 活動スタッフは主体的・明示的に返却できる
  - 完了していなくても返却できる
  - 返却期限は設定しない（物理カードではないため）
- 回収（編集スタッフ主体）
  - 編集スタッフは返却されていない区域を強制回収できる
  - 滞留時は編集スタッフが強制回収で対応する
- 再貸し出し
  - 他のチームが引き受けたり、他のチームに貸し出したりできる
    - 他のチームに貸し出す場合は貸し出し操作を行った編集スタッフが担当者となる

### 担当者

- 便宜的な存在であり、編集スタッフが任意のメンバーから任命できる
- チームに所属しているかどうかは入力補助のレベル（制約ではない）
- チーム内の責任者は必要

### 訪問記録の修正・削除

- 活動スタッフ向けアプリの訪問記録セクションを参照
- 管理アプリからも編集可能（上位互換）
