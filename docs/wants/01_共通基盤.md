# 共通基盤

## データ構造方針

- 全てのデータは木構造とする
- データの競合解決（同期・マージ）はLinkSelfのカバー範囲とする

## 用語定義

- 本仕様における「区域」は運用上の最小単位（枝番レベル）を指す
  - 正確には区域とは枝番の集合だが、メンバーが主に扱うのは枝番レベルであるため「区域」と呼称する
- 枝番の集合としての上位概念は「区域親番」と呼ぶ

## アプリの関係

- 管理メンバー向けアプリは活動メンバー向けアプリの上位互換である
  - 管理アプリから訪問記録の閲覧・編集も可能
  - 管理アプリの地図は編集可能、活動アプリの地図は閲覧・訪問記録用

## 実行環境

- 管理メンバー向けアプリ
  - Windowsアプリ
  - Macアプリ
- 活動メンバー向けアプリ
  - モバイルネイティブ（iOS/Android）を想定
  - LinkSelfはサーバーレスのためWebは想定しない

## データインフラ

- LinkSelfを使用
  - https://github.com/SeijiShii/link-self
  - 完全サーバーレス
    - データはLinkSelfグループ外に出ることはない
- 管理メンバーと訪問活動メンバーは同じグループに所属し、ロールが異なる

### ストレージ（LinkSelf MyDB / SQLite3）

- LinkSelf の `MyDB` を経由して SQLite3 にデータを永続化する
  - SQL（`Exec/Query/QueryRow/Migrate`）と KV（`Put/Get/Delete/List`）の両 API を持ち、書き込みは透過的に同期される
  - DB ファイル配置は LinkSelf 側で `<dataroot>/<DID>/suites/<SuiteID>/data.db` に自動配置（`Config.SuiteID = "jp.home-visit-suite"`）
- 単一プロセスからのみアクセスする想定（Wails アプリ）
- スキーマは起動時に `MyDB.Migrate` で適用する

### 同期スコープ（SyncScope）

各テーブルは `SetSyncScope` で同期範囲を指定する。

| スコープ | 対象テーブル | 同期範囲 |
|---------|-------------|---------|
| `ScopeNetwork` | regions, parent_areas, areas, places, map_*, users, org_groups, member_tags, teams, activities, activity_assignments, visit_records, visit_record_edits, coverages, schedule_periods, scopes, area_availability, requests, invitations, notifications, audit_log | LinkSelf ネットワーク内全メンバー |
| `ScopeDevice` | personal_notes, personal_tags, personal_tag_assignments, my_settings | 同一ユーザーの端末間のみ |

- LinkSelf Group（同期基盤層）と App 内 OrgGroup（運用上の組織）は別レイヤーであり混同しない
- Topic ベースのサブスクリプションでトラフィック最適化（活動メンバーは自分が関与する領域・区域の topic のみ購読）

### データ移行

- 初回起動時に旧 `~/.home-visit-suite/data/*.json`（JSONFileRepository 時代の遺産）が存在すれば MyDB へインポートし、成功時に `.migrated` マーカーを置く

### ネットワークと端末ペアリング

- LinkSelf の `Network` API でネットワーク作成・メンバー招待・脱退を管理する
- 端末ペアリングは `CreatePairingToken` / `CompletePairing`（QR 経由）で同一ユーザーの別端末を追加する
- 自分の DID は `Client.NodeInfo` から取得し、招待・通知の宛先として使う

## マイページ

- 各アプリはメンバーのマイページを持ち、必要な情報や通知を一覧できる
- UIUXは追って検討する

## アプリ設定画面

- 管理・編集メンバー向けアプリに「設定」画面 (`/settings`) を持つ
- 設定値はユーザースコープ（= LinkSelf 個人設定、ScopeDevice、同一ユーザーの端末間で同期）で永続化する
- 初期の設定項目
  - 言語切替（ja / en）。従来サイドバーに置いていた切替は設定画面に統合する
  - ヘルプ表示リセット（個別に「表示しない」にしたツールチップを一括再表示）
- 追加の設定項目は必要に応じてここに集約していく

## 言語・ローカライゼーション

- UI 言語は `ja` と `en` を最初から両方用意する
- 言語設定はユーザースコープ（LinkSelf 個人設定、キー `ui.locale`）で保存する
- 起動時のちらつき防止のため、前回選択した言語を `localStorage` にミラー保存しておき、起動直後の描画はそれを用いる
  - 並行して LinkSelf から取得し、差分があれば切り替える
- 旧実装の `localStorage` locale キーがあれば初回起動時に LinkSelf へ移行する

## ツールチップ基盤（共通 UI）

- 画面右上に表示するツールチップ型のヘルプ基盤を共通で提供する
- i18n キー命名規則: `tips.<画面>.<動作>` (例: `tips.map.polygon.startDraw`)
  - 将来の他画面追加に備え、汎用的な基盤として設計する
- 振る舞い
  - 新しいメッセージは右上にスライドダウンで出現し、約 5 秒後にフェードアウト
  - 連続表示は 1.5 秒間隔のキューで順次投入
  - 同時表示は最大 5 件、超過時は最古から即時消去
  - 新しいメッセージは先頭に積まれ、既存は下へ押し下げる
  - 同一キーが既に表示中の場合は重複投入しない
  - 各メッセージに「このメッセージを表示しない」リンクがあり、クリックで該当キーを永続的に非表示化
- 非表示化したキーは LinkSelf 個人設定 (`ui.hiddenTipKeys`) に保存
- 設定画面の「ヘルプ表示をリセット」で一括解除可能
