# ユーザー管理と権限

## ユーザー管理機能

- LinkSelfグループ内におけるロール
  - 管理者スタッフ
  - 編集スタッフ
  - 活動スタッフ（LinkSelfグループメンバー）
- スタッフ権限は上位互換とする
  - 強い権限を持つスタッフは下位スタッフの権限を包含する
- 管理スタッフ向けアプリには編集スタッフ以上の権限を持つユーザーがログインできる。

## 管理者スタッフ権限

### 招待と任免

- 他の管理者を任免する
  - 管理者は自己罷免不可とする（管理者0人の状態を防止）
  - 罷免は強制即時罷免とする
  - 強権管理者が相互闘争状態になる最悪の状況を想定する必要もあるが、現時点では深くは検討しない
  - 任命
    - 編集スタッフに任命招待を送る。
    - 編集スタッフが受理すると成立する。
  - 罷免
    - 罷免時にレベルを指定する。
      - 編集スタッフへの降格
      - 活動スタッフへの降格
      - LinkSelfグループからの削除
- 編集スタッフの任免
  - 任命
    - 活動スタッフに任命招待を送る
    - 活動スタッフが受理すると成立する
  - 罷免
    - 罷免時にレベルを指定する。
      - 活動スタッフへの降格
      - LinkSelfグループからの削除
- 活動スタッフの任免
  - 任命
    - LinkSelfグループ招待
    - 招待を受理すると成立する
  - 罷免
    - LinkSelfグループからの削除
- 罷免時の進行中データの扱い
  - 活動スタッフへの降格：訪問活動データの担当者としては継続できる
  - LinkSelfグループからの削除：担当者なしになる。未処理タスクも担当者なしになる

### グループ管理

- 管理者はLinkSelf内に複数のグループを作成し、メンバーを割り当てる
  - 例：　100人のメンバーを5つのグループに分けて割り当てる
  - グループは排他的（1人のメンバーは1つのグループにのみ所属）

## 領域・区域の管理権限

- 領域の管理（追加・編集・削除・並べ替え）は管理者専用ページで行う
  - 左サイドバーの「領域管理」からアクセスする
  - 管理者が領域を定義する（名称と記号の指定。この時点では概念上のデータ）
  - 領域の編集はモーダルで名称・記号を変更できる
    - 記号変更時は配下への影響を警告表示する
  - 領域の表示順は上下ボタンで変更する（ドラッグ&ドロップではない）
  - 領域の削除は確認ダイアログで名称と記号の両方を入力させ、一致した場合のみ削除可能とする
  - ポリゴン作成などの実作業は編集スタッフが行う
- 地図ページの右サイドバーでは領域は閲覧のみ
  - 区域親番・区域の追加・編集・削除は右サイドバーで編集スタッフが行う
- 領域内の区域親番・区域の作成は編集スタッフが行う
- 承認フロー
  - 一そろいの領域内区域が完成した時点で管理者が承認する
  - 承認後は編集スタッフ権限では区域・区域親番の削除ができなくなる
  - 削除を伴わない編集（境界ポリゴンの修正など）は承認後も編集スタッフが行える
    - 境界頂点の編集はマップポリゴンライブラリでバリデーションするので不整合にはならない
  - 承認済みの領域に区域を追加する場合は再承認が必要
- 領域・区域親番・区域の削除は管理者のみが可能

## 編集スタッフとグループの関係

- 編集スタッフとグループは別の軸とする
  - 編集スタッフはグループ横断で権限を行使できる

## メンバータグ機能

- 編集スタッフは他のメンバーにタグを付与できる
  - システム内の権限とは厳格に紐づかないが「この人は○○係」というような共通認識になる。
