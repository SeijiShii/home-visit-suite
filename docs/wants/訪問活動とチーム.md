# 訪問活動とチーム

## 訪問活動チーム

- 1人以上のメンバーでチームを組成できる。
  - 活動スタッフ・編集スタッフの両方がチームを組成できる（上位互換）
  - 1回の訪問活動ごとに組成できるし、再利用もできる。
  - チームはグループ横断で組成できる
  - 1人のメンバーが複数のチームに所属できる
- チーム責任者
  - 1人のときはその人が自動的に責任者
  - 2人以上のときは責任者を決める
    - 責任者は適宜変更可能
    - 決めないときは暗黙的に（通常はリスト要素の最初の人）が責任者になる。

## 訪問活動データ

- 訪問活動データ＝1つの区域の貸し出し（チェックアウト）
  - 訪問活動データが区域データを参照する
  - 訪問活動は1つの区域の網羅開始から終了まで1つのデータが紐づく
  - 1つの訪問活動に対して日時ごとに複数の訪問活動チームを紐づけることができる
- ステータス管理
  - 開始前、活動中、完了済み
  - 活動中に返却できる（未完了の状態で返却・回収）
- 返却
  - 担当者が明示的に返却
  - 編集スタッフが強制回収できる
  - 完了していなくても返却・回収できる
  - 返却期限は設定しない（物理カードではないため、滞留時は編集スタッフが強制回収で対応）
  - 他のチームが引き受けたり、他のチームに割り当てたりできる
    - 他のチームに割り当てる場合は割り当て操作を行った編集スタッフが担当者となる
- 担当者
  - 便宜的な存在であり、編集スタッフが任意のメンバーから任命できる
  - チームに所属しているかどうかは入力補助のレベル（制約ではない）
  - チーム内の責任者は必要
- 訪問記録の修正・削除
  - 活動スタッフ向けアプリの訪問記録セクションを参照
  - 管理アプリからも編集可能（上位互換）

## 区域割り当て

- 同じ区域に複数のチームを割り当てることができる
- 編集スタッフは各メンバーに区域を割り当てる
