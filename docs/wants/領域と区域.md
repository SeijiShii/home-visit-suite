# 領域と区域

## 領域

- 活動領域を定義できる
  - 複数指定できる
    - 通常は市区町村単位
      - 成田市
      - 富里市
      - .etc
  - 領域は名称と記号を持つ
    - 記号は通常2~4文字の英大文字
    - 成田市、NRT
    - 富里市、TMS
    - 町名もあり
      - 七栄、NNA
  - 領域の名称と記号は管理者が編集できる
    - 記号を変更した場合、配下の区域親番・区域のID（例: NRT-001-05）が新記号に連鎖更新される
    - 更新順序は子→親（Area→ParentArea→Region）で参照整合性を保つ

## 区域親番と区域

- 活動領域を区域親番として分割できる
  - 100枚程度の区域親番
  - 区域親番は番号と名称を持つ
    - 加良部1丁目
  - 識別子の体系：`領域-区域親番-区域`
    - 例：NRT-001-05
      - NRT = 領域（成田市）、001 = 区域親番（加良部1丁目）、05 = 区域
    - 表示上は視認性を考慮し `NRT-001-05　加良部1丁目` とする
      - 論理的には `NRT-001-05　加良部1丁目-05` だが区域親番名に区域番号を重複表示しない
  - 区域親番は区域（旧称：枝番）に分割できる
    - 10枚程度の区域カード
    - 1つの区域は50~100件程度の訪問先（家やアパート・マンションの部屋）を含む
