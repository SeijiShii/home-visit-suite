# 領域と区域

## 領域

- 活動領域を定義できる
  - 複数指定できる
    - 通常は市区町村単位
      - 成田市
      - 富里市
      - .etc
  - 領域は名称と記号を持つ
    - 記号は通常2~4文字の英大文字
    - 成田市、NRT
    - 富里市、TMS
    - 町名もあり
      - 七栄、NNA
  - 領域の名称と記号は管理者が編集できる
    - 記号を変更した場合、配下の区域親番・区域のID（例: NRT-001-05）が新記号に連鎖更新される
    - 更新順序は子→親（Area→ParentArea→Region）で参照整合性を保つ

## 区域親番数の管理

- 区域親番数は領域の作成・編集時に指定する
  - 領域作成時: 初期の区域親番数を入力（0以上）
  - 領域編集時: 区域親番数を変更できる
- 区域地図画面（AreaTree）からは区域親番の追加・削除を行わない
  - 親番の名前編集と区域の追加・削除は地図画面で行う
- 区域親番数を増やす場合
  - 不足分の区域親番を自動生成（連番、デフォルト名付き）
- 区域親番数を減らす場合
  - 末尾（番号が大きい）の区域親番から削除する
  - 削除対象の区域親番に所属する区域も削除する
  - 削除される区域に紐づいているポリゴンは「区域紐づきなし」になる（ポリゴン自体は残る）
  - 減少時は確認ダイアログを表示する

## 区域親番と区域

- 活動領域を区域親番として分割できる
  - 100枚程度の区域親番
  - 区域親番は番号と名称を持つ
    - 加良部1丁目
  - 識別子の体系：`領域-区域親番-区域`
    - 例：NRT-001-05
      - NRT = 領域（成田市）、001 = 区域親番（加良部1丁目）、05 = 区域
    - 表示上は視認性を考慮し `NRT-001-05　加良部1丁目` とする
      - 論理的には `NRT-001-05　加良部1丁目-05` だが区域親番名に区域番号を重複表示しない
  - 区域親番は区域（旧称：枝番）に分割できる
    - 10枚程度の区域カード
    - 1つの区域は50~100件程度の訪問先（家やアパート・マンションの部屋）を含む
