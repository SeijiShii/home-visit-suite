# 活動メンバー向けアプリ

- 地図主体のアプリ

## 区域

- 活動メンバーも区域一覧を閲覧できる
- 活動メンバーがデータを入力できるのは自分が貸し出し・持ち出しでチェックアウトしている区域

## 訪問記録

- 活動メンバーが訪問内容や訪問場所の状況を記録する。
- 入力項目
  - 日時（ダイアログ起動時に自動入力、手動編集可）
  - 訪問ステータス（スピナー選択）
  - 訪問メモ（個人専用、自由記述）
  - 自前のタグ（個人用、検索・管理に利用）
- 共有/非共有の区分
  - **ネットワーク共有**: 日時・訪問ステータス（網羅状況の把握、再訪事故防止のため）
  - **個人専用（非共有）**: 訪問メモ、個人タグ
  - 共有情報も個人情報保護との兼ね合いを配慮する

### 訪問ステータス（スピナー選択肢）

戸建て・部屋いずれの訪問記録ダイアログでも以下の 5 択から選ぶ（集合住宅単位での記録は持たない。詳細は `## 訪問記録ダイアログ` 節）。

| ステータス | 性質 | 追加入力 | 後続処理 |
|---|---|---|---|
| 会えた | ネットワーク共有 | なし | 網羅判定、再訪間隔の参考データ |
| 留守 | ネットワーク共有 | なし | 網羅判定、他メンバーが再網羅可能 |
| 空き家（入居の可能性あり） | ネットワーク共有 | なし | 一時的な空室として扱う。再訪対象 |
| 空き家（廃屋）または更地 | ネットワーク共有 + 申請 | テキスト入力（必須／状況補足） | **地図情報更新申請**として編集メンバーのタスクへ登録 |
| 訪問を望まない | ネットワーク共有 + 申請 | テキスト入力（必須／状況補足） | **訪問拒否宅報告**として編集メンバーのタスクへ登録 |

- 申請を伴うステータス（「空き家（廃屋）または更地」「訪問を望まない」）は、選択直後にテキスト入力ダイアログを開く。テキスト入力をキャンセルした場合はステータス選択も無効に戻す。
- 申請タスクの処理仕様（受理/却下、データ反映フロー等）は次期スコープで設計する（`07_通知と申請.md`）。

### 訪問記録のデータ構造

- 場所データ（NULL可）：場所モデルへの参照
- 座標（NULL可）：場所モデル未登録の地点にも記録可能
- 場所追加申請は場所＝NULLで座標必須
- **場所も座標も持たない訪問記録**は本スコープでは扱わない（`09_継続的検討事項.md` 参照、別途検討）

### 訪問記録の修正・削除

- 活動メンバーは誤入力した訪問記録を修正・削除できる
- 管理アプリからも編集可能（上位互換）
- 編集履歴を保持する

## 訪問記録ダイアログ

地図上の場所アイコンをタップして開くダイアログ。戸建て／集合住宅／部屋（集合住宅内）で構成が異なる。

### 戸建て訪問ダイアログ（`Place.Type == "house"`）

- 起動: 地図上の戸建てアイコン（青）をタップ
- 上部: 場所の `Label`（名前、未入力なら「名前なし」）と `Address`
- 入力エリア
  - 日時（自動入力、編集可）
  - 訪問ステータス（スピナー、5択は前述）
  - 訪問メモ（テキスト入力、個人専用）
- 参考情報「最近会えた日付（ネットワーク全体）」
  - 対象は **「会えた」記録のみ**
  - 表示は **日付のみ**（例: `2026-04-15 に会えた`）。時刻・訪問者名は表示しない
  - 経過日数で色分け
    - **1ヶ月以内: 赤文字**（再訪間隔が近すぎる警告）
    - **半年以内: オレンジ文字**
    - **それ以上: 黒文字**
  - ネットワーク全体で「会えた」記録が一度もない場合は「会えた記録なし」と表示
- 自分の訪問履歴セクション（折りたたみ可、初期は折りたたみ）
  - 同一場所に対する自分の訪問記録を時系列降順で一覧表示
  - 履歴がなければ「自分の訪問履歴なし」と表示
- ダイアログ下部「**場所情報の修正を申請**」ボタン
  - クリックでテキスト入力ダイアログを開く（戸建て・集合住宅で UI 共通）
  - 入力内容は **場所情報修正申請**として編集メンバーのタスクへ登録（次期スコープ、現フェーズはデータ保持のみ）
  - 申請であり情報の即時更新ではない
- 保存・キャンセル
  - 保存: 訪問記録を作成。スピナーが申請を伴うステータスの場合、テキスト入力後に同時に申請データも作成
  - キャンセル: すべて破棄

### 集合住宅訪問ダイアログ（`Place.Type == "building"`）

- 起動: 地図上の集合住宅アイコン（オレンジ）をタップ
- 集合住宅自体には**スピナーを持たない**（建物単位の訪問ステータスは記録しない）
- 上部: `Label`（名前）と `Address`
- **Description 表示**: `Place.Description` が非空（length > 0）の場合、上部の見やすい位置に表示する
  - 編集メンバーが集合住宅編集ダイアログで設定する補足情報（注意事項、訪問時の留意点など）
  - `Description` フィールド自体の仕様は `03_地図機能.md` の「集合住宅の追加・編集」節に追記
- 部屋番号一覧
  - 当該集合住宅の Room 一覧を `SortOrder` 昇順で表示
  - 行表示: 部屋番号（`DisplayName`）+ 自分の最終訪問日（あれば、戸建てと同じ色分けルール）
  - 行タップで **部屋訪問ダイアログ**（戸建て訪問ダイアログと同等構成）を開く
- ダイアログ下部「**場所情報の修正を申請**」ボタン
  - クリックでテキスト入力ダイアログを開く
  - 用途例: 部屋番号の修正、建物全体が立ち入り禁止になった、入口の変更など
  - 入力内容は **場所情報修正申請**として編集メンバーのタスクへ登録

### 部屋訪問ダイアログ（`Place.Type == "room"`）

- 起動: 集合住宅訪問ダイアログ内の部屋行タップ
- 構成は戸建て訪問ダイアログと同一（スピナー、メモ、参考情報、自分の履歴、修正申請ボタン）
- 上部表示は親建物の名前と部屋番号（例: `○○マンション 101号室`）

### 場所作成申請ダイアログ

- 起動: 訪問記録画面の地図上で **空白部分をロングタップ**（場所アイコン上のロングタップは無視）
- 用途: 活動中に未登録の建物を見つけた場合の追加申請
- 入力項目
  - 種別選択: **家 / 集合住宅 / その他**
  - テキスト入力（補足情報、必須）
  - 集合住宅でも部屋番号など詳細は入力しない（編集メンバーが現地調査時に設定）
  - 「その他」は店舗・公共施設等を想定（種別の正式定義は別途検討、`09_継続的検討事項.md`）
- 座標はロングタップ位置から自動取得
- 保存で **場所作成申請**として編集メンバーのタスクへ登録（場所＝NULL、座標必須）
- ロングタップによる誤操作防止 UX は別途検討（`09_継続的検討事項.md`）

## 訪問記録画面

訪問ダイアログを起動するための地図画面。管理・編集メンバー向けアプリ (Wails) では左サイドバーに独立メニューとして配置する。

### 入口

- 管理アプリ (Wails): **左サイドバーに「訪問記録」メニュー**を新設
  - 利用可能ロール: **全ロール**（管理者・編集メンバー・活動メンバー、上位互換原則）
- モバイルアプリ: ナビゲーション仕様はモバイル UI 設計時に別途定義

### 起動後の遷移

- **最終形**: 「現在チェックアウト中の区域」一覧を表示し、区域選択で当該区域の地図画面に遷移する
- **Phase 1 暫定**: チェックアウトモデルが未設計のため、ダミーの 1 画面（または直接遷移）として、ポリゴン紐付け済みの区域 `NRT-001-01` の地図に固定遷移する
  - チェックアウト関連の本実装着手時に最終形へ移行する

### 地図 UI

- 表示密度は `03_地図機能.md` の **区域詳細編集モードと同等**（場所アイコン、通し番号バッジ、対象区域輪郭の濃色表示など）
- ただし**編集系操作は一切提供しない**（場所追加・移動・削除、ポリゴン編集、コンテキストメニュー等は無効）
- 場所一覧サイドパネル（区域詳細編集の右パネル）も訪問記録画面では設けない（モバイル想定のためフルスクリーン地図を優先）
  - 必要に応じて将来追加検討
- 共通の地図ビュー部品を再利用するか専用部品とするかは実装フェーズで判断する

### 操作仕様

- **タップ**（マウスクリック / 単指タップ）
  - 戸建てアイコン → 戸建て訪問ダイアログ
  - 集合住宅アイコン → 集合住宅訪問ダイアログ
  - 空白タップ → 何もしない（誤操作で記録を作らないため）
- **ロングタップ**（モバイル長押し / マウス右クリック）
  - 場所アイコン上 → 何もしない（編集機能なし）
  - **空白部分** → 場所作成申請ダイアログ
  - ロングタップは編集系の代替には使わない（モバイル UX 低下のため原則使わない方針の例外として、場所作成申請のみ採用）
- **マップ操作**: パン・ピンチズームは標準。ズーム制限は区域詳細編集と同等

## 最新の状況

- 状況の変化
  - 建物が取り壊されて更地になっている
  - 以前は住んでいた家が廃屋、空き室になっている。
  - 更地に新築
  - 1つまたは複数の更地または戸建てが集合住宅に建て替わった。
  - 外国人居住情報
    - 言語によっては専従メンバーが訪問できるように情報を切り分ける
- 簡易な情報は活動メンバーが記入できる
  - 表札名が空欄だったところに入力、変更など
  - 集合住宅が新築されたときなどは「地図情報更新申請」を投稿できる
    - 申請を受けて編集メンバーが詳細情報を調査する
- 場所データの追加
  - 活動メンバーが新築等で場所モデルにない地点を追加できる
  - 編集メンバーによる事後承認とし、申請としてタスク化する（申請タスクと同一の仕組み）
  - 不適切な場所データ（建物が建たない場所など）は編集メンバーが確認して無効化できる
    - 無効化時、その場所に紐づく訪問記録は編集メンバー権限で破棄される（確認UIが必要）
- 訪問をしないお家
  - 住人または関係者の意思表示、または危険な状況により一定期間訪問しないほうが良い場所を報告する
  - 編集メンバーはその情報を精査して区域に反映する
    - このデータ反映は権限者が行う
  - 訪問不可状態の解除
    - 編集メンバーは地図上で訪問拒否宅の詳細を確認できる。
      - 別途規定するガイドラインに則り訪問再開の可否を判断し編集できる。
