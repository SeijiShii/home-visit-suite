# 活動スタッフ向けアプリ

- 地図主体のアプリ

## 区域

- 活動スタッフも区域一覧を閲覧できる
- 活動スタッフがデータを入力できるのは自分が貸し出し・持ち出しでチェックアウトしている区域

## 訪問記録

- 活動スタッフが訪問内容や訪問場所の状況を記録する。
  - 日時
  - 会えたか、留守か
    - 留守情報はLinkSelfグループ全体に共有して他のメンバーがその場所を再網羅できることを前提とする。
    - 個人情報保護との兼ね合いを配慮する
  - 活動スタッフは訪問時の家の人の反応などの情報を記述できる
    - これらは個人情報に当たるので他のメンバーに共有されない
    - 自前のタグを登録して訪問記録の検索や管理ができる
- 訪問記録のデータ構造
  - 場所データ（NULL可）：場所モデルへの参照
  - 座標（NULL可）：場所モデル未登録の地点にも記録可能
  - 場所も座標も持たない訪問記録も存在しうるが、申請データにはならない（活動スタッフの個人データ）
  - 場所追加申請は場所＝NULLで座標必須
- 訪問記録の修正・削除
  - 活動スタッフは誤入力した訪問記録を修正・削除できる
  - 管理アプリからも編集可能（上位互換）
  - 編集履歴を保持する

## 最新の状況

- 状況の変化
  - 建物が取り壊されて更地になっている
  - 以前は住んでいた家が廃屋、空き室になっている。
  - 更地に新築
  - 1つまたは複数の更地または戸建てが集合住宅に建て替わった。
  - 外国人居住情報
    - 言語によっては専従スタッフが訪問できるように情報を切り分ける
- 簡易な情報は活動スタッフが記入できる
  - 表札名が空欄だったところに入力、変更など
  - 集合住宅が新築されたときなどは「地図情報更新申請」を投稿できる
    - 申請を受けて編集スタッフが詳細情報を調査する
- 場所データの追加
  - 活動スタッフが新築等で場所モデルにない地点を追加できる
  - 編集スタッフによる事後承認とし、申請としてタスク化する（申請タスクと同一の仕組み）
  - 不適切な場所データ（建物が建たない場所など）は編集スタッフが確認して無効化できる
    - 無効化時、その場所に紐づく訪問記録は編集スタッフ権限で破棄される（確認UIが必要）
- 訪問をしないお家
  - 住人または関係者の意思表示、または危険な状況により一定期間訪問しないほうが良い場所を報告する
  - 編集スタッフはその情報を精査して区域に反映する
    - このデータ反映は権限者が行う
  - 訪問不可状態の解除
    - 編集スタッフは地図上で訪問拒否宅の詳細を確認できる。
      - 別途規定するガイドラインに則り訪問再開の可否を判断し編集できる。
