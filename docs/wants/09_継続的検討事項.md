# 継続的検討事項

## 権限（解決済み）

- 編集メンバーが稼働できないときに活動メンバーだけで区域を取得できるように限定的な権限範囲の追加が必要か
  - → 「持ち出し可能区域」の仕組みで解決。編集メンバーが事前に持ち出し可能区域を設定しておけば、活動メンバーが自主的に区域を取得できる
  - 詳細は「05_訪問活動とチーム.md — 区域の貸し出し（チェックアウト）モデル」を参照

## 区域系画面の権限切り分け

- 区域編集・区域詳細編集など区域系画面の利用可能ロールを別途整理する
- 例: 区域詳細編集を編集メンバー以上に限定するか、活動メンバーは閲覧のみとするか等
- 訪問記録画面（`08_活動メンバー向けアプリ.md`）は全ロール利用可で確定済み

## 場所も座標もない訪問記録の扱い

- 旧仕様で「場所も座標も持たない訪問記録」を活動メンバーの個人データとして許容していたが、本フェーズではスコープ外とした
- 必要性（GPS取得失敗時の救済、純粋な個人メモ用途等）と、訪問記録メニューからの起動経路を別途設計する

## 場所作成申請の「その他」種別の定義

- 場所作成申請ダイアログの種別選択肢「その他」（家・集合住宅以外）の正式定義を別途検討
- 想定される対象: 店舗、公共施設、事業所、寺社、学校など
- 「その他」をさらに細分化するか、自由記述に委ねるかも論点

## ロングタップによる場所作成申請の誤操作防止 UX

- 訪問記録画面では原則ロングタップを使わない方針に対して、場所作成申請のみ例外的にロングタップで起動する
- モバイルでの誤操作（地図操作中の意図しないロングタップ）を防ぐ UX を別途検討
- 候補: 確認ダイアログを必ず挟む、長押し時間を通常より長く設定、起動前にトースト/ヘルプを出す等
