# 網羅管理

## 網羅活動データ

- 区域親番単位で「網羅活動」データを持つ
  - 訪問活動データ（区域単位）を集約した上位データ
  - 網羅活動は網羅予定と紐づく
- 作成・管理権限
  - 網羅活動データは編集メンバーが予定策定時に作成する
  - 編集メンバーすべてに編集権限があるが、実質的には管理者から一任されたメンバーが行う（タグ管理レベル）
  - 予定策定・編集も承認フローとする
  - 編集履歴は監査ログに記録される
- 完了パーセンテージ
  - 実体完了パーセンテージ：全区域の訪問活動完了状況から算出
  - ステータス上の完了パーセンテージ：運用上の恣意的な完了判定
    - 例：実体は50%だがステータスとして完了とする場合がある
  - 理想は全区域の訪問活動完了をもって区域親番の完了だが、運用上は恣意的に完了させることもできる

## 進捗管理

- 訪問活動データを集計
- 指定した期間内における網羅状況を確認できる
  - 例：　9/1からの3か月間でいくつの区域親番、全体の何パーセント網羅したか
- 地図上で区域親番ごとの網羅頻度などをヒートマップ表示できる

## 網羅予定管理

編集メンバーが戦略的に網羅予定を立てる（承認フロー対象）。

### 予定期間（SchedulePeriod）

- 予定期間を定義する（例: 2026年1月〜2月）
- 予定期間は名前・開始日・終了日を持つ
- **重複不可**: 複数の予定期間が時間的に重複することはできない
- 承認フロー対象

### スコープ（Scope）

- 予定期間を親として、複数のスコープを設定する
- **グループスコープ**: メンバーグループからコピーされ、グループ名がそのままスコープ名になる（グループと1:1対応）
- **非グループスコープ**: グループに紐付かない自由定義のスコープ（例: 休日活動、集合住宅メイン）
- それぞれのスコープに区域親番を追加する
- **排他制約**: 1つの予定期間内で、区域親番が所属できるスコープは1つのみ
- **期間横断**: 異なる予定期間間では、1つの区域親番が複数のスコープに所属できる

### 区域の可用性設定

- スコープの中で区域の可用性を設定する
  - **貸し出し可能区域**: スコープに含まれる区域親番の配下から、ある期間に使用する区域を貸し出し可能に設定する
  - **持ち出し可能区域**: 貸し出し可能区域の中から、活動メンバーが自主的に取得できる区域を区域単位で設定する
  - 詳細は「05_訪問活動とチーム.md — 区域の貸し出し（チェックアウト）モデル」を参照

### データモデル

```
SchedulePeriod（予定期間）
├── id, name, startDate, endDate, approved
└── Scope（スコープ）※複数
    ├── id, name, groupId（グループスコープ時のみ）
    ├── parentAreaIds[]（区域親番リスト）
    └── AreaAvailability（区域可用性）※複数
        └── areaId, type(lendable|self_take), startDate, endDate
```

- 旧 CoveragePlan は SchedulePeriod + Scope に分離
- Activity.coveragePlanId → Activity.scopeId に変更
- AreaAvailability.coveragePlanId → AreaAvailability.scopeId に変更

## 進捗把握

- 予定と進捗を照合することで進捗管理ができる
- 予定期間の開始日・終了日に基づいて進捗を追跡する
