# 網羅管理

## 網羅活動データ

- 区域親番単位で「網羅活動」データを持つ
  - 訪問活動データ（区域単位）を集約した上位データ
  - 網羅活動は網羅予定と紐づく
- 作成・管理権限
  - 網羅活動データは編集スタッフが予定策定時に作成する
  - 編集スタッフすべてに編集権限があるが、実質的には管理者から一任されたスタッフが行う（タグ管理レベル）
  - 予定策定・編集も承認フローとする
  - 編集履歴は監査ログに記録される
- 完了パーセンテージ
  - 実体完了パーセンテージ：全区域の訪問活動完了状況から算出
  - ステータス上の完了パーセンテージ：運用上の恣意的な完了判定
    - 例：実体は50%だがステータスとして完了とする場合がある
  - 理想は全区域の訪問活動完了をもって区域親番の完了だが、運用上は恣意的に完了させることもできる

## 進捗管理

- 訪問活動データを集計
- 指定した期間内における網羅状況を確認できる
  - 例：　9/1からの3か月間でいくつの区域親番、全体の何パーセント網羅したか
- 地図上で区域親番ごとの網羅頻度などをヒートマップ表示できる

## 網羅予定管理

- 編集スタッフが戦略的に網羅予定を立てる（承認フロー対象）
- グループごとに期間を指定して網羅予定を区域親番単位で割り当てる
  - 例：　Aグループに3~4月に10の区域親番を指定して割り当てる
- 網羅予定の中で区域の可用性を設定する
  - **貸出可能区域**: 網羅予定に含まれる区域親番の配下から、ある期間に使用する区域を貸出可能に設定する
  - **持ち出し可能区域**: 貸出可能区域の中から、活動スタッフが自主的に取得できる区域を区域単位で設定する
  - 詳細は「05_訪問活動とチーム.md — 区域の貸し出し（チェックアウト）モデル」を参照

## 進捗把握

- 予定と進捗を照合することで進捗管理ができる
- 網羅予定期間と開始日、終了日
