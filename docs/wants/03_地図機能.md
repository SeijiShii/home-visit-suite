# 地図機能

## 画面構成

- 編集メンバー向けには「区域編集」画面（旧称: 区域地図）を提供する
  - ポリゴンの作成・編集・紐付け、区域親番/区域の編集を行う
- 訪問スタッフ向けには別途「区域地図」画面を後日追加する（閲覧+訪問記録用）
- 区域編集画面の中から「区域詳細編集」モードに遷移できる（後述）

## ポリゴンクリック操作

- ポリゴン**シングルクリック** → ポリゴン編集モードに入る（頂点ドラッグ可）
- ポリゴン**ダブルクリック** → 区域詳細編集モードに入る（紐付け済み区域のみ）
- ポリゴン**右クリック** → コンテキストメニュー
  - ポリゴン編集
  - 区域詳細編集（紐付け済みのときのみ活性）
  - 家を追加（区域詳細編集モード中のみ表示）
- 空白クリックで選択解除（ポリゴンクリックはマップへ伝播させない）
- 区域一覧の三点メニューにも「区域詳細編集」を追加し、行ダブルクリックでも遷移できる

## 区域詳細編集モード

- 起動経路（3 つ、いずれも同じ画面に遷移）
  1. 区域一覧の三点メニューから「区域詳細編集」
  2. 区域一覧の区域行ダブルクリック
  3. 地図上のポリゴンダブルクリック
- 起動条件: ポリゴン紐付け済みの区域のみ
- 画面構成
  - 上部バー: ← 戻る ボタン + 区域名（`{領域記号}-{親番}-{区域番号}`） + 右端に **場所一覧パネルの開閉ボタン**（chevron アイコン）
  - 区域編集画面の右サイドバー（ポリゴン一覧 / 区域一覧）は非表示
  - **場所一覧サイドパネル**（右側、既定で開く／後述「場所一覧サイドパネル」節参照）
  - 地図は残り幅に追従（パネル開閉時に Leaflet `invalidateSize()` を呼び再レイアウト）
- 描画ルール
  - 編集対象区域の輪郭: 濃色
  - 隣接区域の輪郭: 薄色
  - 塗りつぶしなし
  - 「隣接」の定義: 編集対象区域の中心から半径 N km 内に中心があるすべての区域
- ズーム制限
  - 詳細編集中は最小ズームレベルを制限し、半径 N km 程度が画面に収まる範囲を維持
  - N km は設定画面 (`/settings`) で変更可能（既定 2.5km、キー: `ui.areaDetailRadiusKm`）
- 場所操作
  - ポリゴン内右クリック → コンテキストメニュー「家を追加」「集合住宅を追加」
    - 「家を追加」選択後、入力ダイアログが開き「住所」「名前」を任意入力（空のまま保存可能）→ 保存で `Place` (`Type="house"`) を作成
    - 「集合住宅を追加」選択後、集合住宅用の入力ダイアログが開く（後述「集合住宅の追加・編集」節）
    - 5m 以内に削除済み場所がある場合は先に復元確認ダイアログ、その後に入力ダイアログを表示（**集合住宅追加時は復元確認なし**）
  - 場所アイコン右クリック → 「場所編集」「移動」「削除」
    - 場所編集: 戸建てと集合住宅で別のダイアログを開く
      - 戸建て (`house`): 現在の名前・住所を初期値にした編集ダイアログ（作成ダイアログと同一 UI）。保存で `Place.Label` / `Place.Address` を更新
      - 集合住宅 (`building`): 集合住宅編集ダイアログ（後述「集合住宅の追加・編集」節）
    - 移動: 選択直後からマウス追従（ボタンは押さない）。クリックで確定 / Esc キャンセル。移動中は地図ドラッグを無効化。集合住宅でも同じ挙動（部屋の `Coord` は移動しない）
    - 削除:
      - 戸建て: 確認ダイアログ → 論理削除
      - 集合住宅: 確認ダイアログ（「この集合住宅と N 部屋を削除しますか？」）→ **Building と全 Room を同時に論理削除（カスケード）**
  - 追加・移動の対象座標が**編集中の区域ポリゴン外**の場合は処理を中止し、画面上部に赤いエラーバナーを 4 秒表示する
    - メッセージ: 「場所の追加または移動は編集中の区域内でのみ可能です」
- 場所アイコン
  - 種別ごとに色を変えた円形（**house=青 `#2563eb`、building=オレンジ `#ea580c`**）
  - Phase 1 では Room 個別のアイコンは地図に表示しない（Room は座標 `(0, 0)` のダミーで保持、地図描画対象外）
  - 将来 Room を地図表示する必要が出た場合は別色（例: 緑）を割り当てる
  - 半径はズームレベル連動 (8〜14px)
  - モバイル環境でも無理なくタップ可能なサイズ
  - ホバー時にツールチップで `Label`（名前）を表示。未入力の場合は「名前なし」
  - 円形アイコンに **通し番号バッジ**を重ねて表示する（`Place.SortOrder + 1`、区域内で 1 始まり）
  - 場所一覧パネルで選択中の行に対応するアイコンは強調表示する
- 区域編集モード（親モード）では場所アイコンは表示しない

## 集合住宅の追加・編集

- データ構造
  - 集合住宅は `Place` (`Type="building"`) と、その子として `Place` (`Type="room"`, `ParentID=<BuildingID>`) を複数持つ
  - Room の `Coord` は `(0, 0)`（ダミー。地図上には個別描画しない）
  - Room の `Label` は使用しない（空文字）。部屋番号は `Place.DisplayName` に格納する
  - Room の `SortOrder` は同一 Building 内の並び順（0 始まり）
- 作成ダイアログ (`BuildingEditDialog`、新規モード)
  - 入力項目
    - 住所（任意）
    - 名前（任意）
    - 部屋番号一覧
      - 各行: 部屋番号入力欄（空欄可）+ `×` 削除ボタン
      - 初期状態は 1 行（空欄）
      - 行数の最小は 1（最後の 1 行は `×` が無効）
      - 行の下部に `[+1] [+5] [+10]` ボタン（押下で空行を N 行追加）
      - 行の **D&D 並び替え** に対応
  - 保存処理
    - Building を 1 件 `savePlace` で作成
    - Building の戻り `id` を使い、Room を行数分 `savePlace` で作成
    - Room の `DisplayName` は入力値、`SortOrder` は行順で 0..N-1、`Coord` は `(0, 0)`、`Label` は空
    - Building の `SortOrder` は区域内の既存 Place 最大値 + 1
  - キャンセル
    - 入力内容は破棄し何も保存しない
- 編集ダイアログ (`BuildingEditDialog`、編集モード)
  - 作成ダイアログとほぼ同じ UI。既存 Room を `SortOrder` 昇順で行として表示
  - 行操作
    - 新規追加行の `×`: 即座に行削除（確認なし）
    - 既存 Room 行の `×`: 確認ダイアログ（「この部屋を削除しますか？」）→ Save 時に `deletePlace` で論理削除
    - D&D 並び替え: 保存時に全 Room の `SortOrder` を行順で再採番
    - 部屋番号（`DisplayName`）編集: 保存時に `savePlace` で反映
  - 保存処理は差分適用（追加 / 更新 / 削除）を一括でサービス呼び出し
- トリガー
  - 右クリックコンテキストメニュー「集合住宅を追加」→ 作成ダイアログ
  - 集合住宅アイコン右クリック → 「場所編集」→ 編集ダイアログ
  - 場所一覧パネルの集合住宅行ダブルクリック → 編集ダイアログ
- カスケード削除
  - Building の `deletePlace` 時、同じ `ParentID` を持つ全 Room（`DeletedAt == null`）も `DeletedAt` を立てて論理削除する
  - 確認ダイアログメッセージには対象部屋数を含める

## 場所一覧サイドパネル（区域詳細編集モード）

- 区域詳細編集モードの右側に、対象区域に属する場所の一覧を表示する収納可能なサイドパネルを設ける
- 既定状態は「開」、幅は 320px 目安
- 開閉操作
  - 上部バー右端の chevron アイコンボタンで開閉する
  - 開閉状態は LinkSelf 個人設定に保存（キー: `ui.areaDetailPlaceListOpen`、端末間同期）
  - 閉じているときは地図が全幅に広がる（パネル部分は 0 幅）
- 表示対象
  - 対象 `areaId` に属する `Place` のうち `DeletedAt == null` かつ `ParentID == ""` のもの
  - **戸建て (`house`) と集合住宅 (`building`) をフラットに同一リストに表示**（`SortOrder` を共有、通し番号も通し）
  - Room (`Type == "room"`) は一覧パネルには表示しない（集合住宅の編集ダイアログ内で管理）
- 行表示項目（左から）
  - 通し番号バッジ（`SortOrder + 1`、1 始まり）
  - 種別アイコン（house=青、building=オレンジ）
  - 1 行目: `Label`（未入力時は「名前なし」）
  - 2 行目:
    - 戸建て: `Address`（小さめのフォント、未入力時は非表示）
    - 集合住宅: `🏢 N部屋` 形式で子 Room 件数（`DeletedAt == null` のみ）を併記。`Address` があれば続けて表示
- 並び順と通し番号
  - 並び順は `Place.SortOrder` の昇順
  - 通し番号 = `SortOrder + 1`（表示上のみの換算。永続化値は 0 始まり）
- 並び替え（ドラッグ＆ドロップ）
  - 行を縦方向にドラッグして順序を入れ替えられる
  - 確定時、対象区域内のすべての場所に 0..N-1 の新しい `SortOrder` を再採番し、順次 `PlaceService.savePlace` で保存する（将来的にバルク API 検討）
  - 番号は並び替え完了時に即座に一覧と地図アイコンのバッジに反映される
  - アクセシビリティのためキーボード操作（上下矢印＋Space/Enter）にも対応する
- 初期 `SortOrder` の確定
  - 区域詳細編集モードの初回起動時、対象区域内のすべての場所の `SortOrder` が 0 の場合、**作成日時 (`CreatedAt`) の昇順**で 0..N-1 を採番し保存する
  - 以後、既存場所の `SortOrder` は削除・並び替え以外で変更しない
- 行クリック時の地図連動
  - 行（背景部分）クリック: パネル上で選択状態にし、**地図を当該場所の座標までパン移動**する（`map.panTo`、ズームレベルは維持）
  - 行内の三点メニュー等の操作要素は別途（クリックイベントは `stopPropagation`）
- 行ダブルクリック
  - 戸建て行: 「場所編集」ダイアログ（`AddPlaceInputDialog` 編集モード）を開く
  - 集合住宅行: 集合住宅編集ダイアログ (`BuildingEditDialog`) を開く
- 新規追加・削除との連動
  - 「家を追加」「集合住宅を追加」で `Place` が作成された場合、`SortOrder` は既存最大値 + 1 として末尾に追加される
  - 削除（論理削除）した場合、一覧から即時除外し、残りの場所の `SortOrder` は詰め直さない（番号の欠落は許容）
  - ただし次回並び替え操作時には全体を 0..N-1 で再採番する

## 場所の論理削除と訪問記録の紐付け

- 場所 (`Place`) は物理削除せず、`DeletedAt` を立てる論理削除
- 場所を追加・移動して確定する際、座標から半径 5m 以内に削除済みの場所がある場合、
  「過去のこの地点の場所の訪問記録と紐づけますか」確認ダイアログを表示する
- 「はい」を選んだ場合、新しい場所に `RestoredFromID`（旧 PlaceID への参照）を持たせる
- 訪問記録自体の付け替えは、訪問記録機能の設計時に対応する（Phase 1 ではフラグ保持のみ）

- 領域、区域親番、区域をポリゴンとして作成できる
  - 地図ポリゴン編集ライブラリ
    - https://github.com/SeijiShii/map-polygon-editor
    - レポジトリ作成者が同一なので不足する機能は追加可能
- 地図タイルはOpen Street Mapまたは日本国内向けには国土地理院地図を使用
  - OSMが理想だが、細かい住宅建物情報などが欠落しているので地理院地図を使用する
    - 参考デモアプリ
      - https://github.com/SeijiShii/map-polygon-editor-demo
      - 要件が絞り切れていないため、機能は非常に限定的
- ポリゴンの整合性検証（重なり・隙間の検出・防止）はマップポリゴンライブラリの責任範囲

## ポリゴン一覧での区域紐付け・紐付け解除

- 区域と紐づいていないポリゴン
  - リンクアイコンボタン（鎖マーク）を表示
  - クリックするとダイアログが開く
    - ダイアログ内は区域一覧と同じツリービュー
    - ポリゴンと紐づいていない区域を選択できる
    - 紐づけ済みの区域はグレーアウトして不活性
    - 区域をクリックすると「区域○○と紐づけました」というトーストが出てダイアログが閉じる
- 区域と紐づいているポリゴン
  - リンク解除ボタン（鎖カットマーク）を表示
  - クリックすると「区域○○との紐づけを解除しますか？」確認ダイアログ
  - 解除ボタンクリックで解除

## 区域一覧（AreaTree）でのポリゴン紐付け表示と解除

- ポリゴンが紐づいている区域にはグリーンのポリゴンアイコンを表示
- ポリゴンアイコンの横に三点リーダーメニューボタンを表示
  - クリックすると「ポリゴン紐づけ解除」メニューが表示される
  - メニュー選択で「区域○○との紐づけを解除しますか？」確認ダイアログ
  - 解除ボタンクリックで解除
- ポリゴン描画開始ボタンは区域一覧には表示しない（描画はポリゴン一覧側から行う）

## 住宅マーカーの区域帰属

- マーカー座標がポリゴン内にある状態をもって区域に所属する場所とする
  - ポリゴンの隣地に建物が建った場合などはポリゴンの編集が必要
- 訪問活動中にポリゴン外に建物を見つけた場合
  - 訪問活動中は地図アプリ上で「どの区域で活動中」という情報を持つので、申請情報に区域を持たせる
  - ポリゴン外でも申請できるようにする
  - 編集メンバーが他のポリゴンに帰属すると判断する場合は編集時に変更できる

## ポリゴンの同時編集

- ポリゴンネットワークは頂点・辺・ポリゴンをエンティティ単位で LinkSelf MyDB に格納する
  - テーブル: `map_vertices` / `map_edges` / `map_polygons`（いずれも `ScopeNetwork`）
  - 旧来の「ネットワーク全体を 1 つの JSON として読み書き」する方式（`map_networks` 単一 BLOB）は廃止
- map-polygon-editor の `StorageAdapter` はレコード単位の `putVertex` / `deleteVertex` / `putEdge` / `deleteEdge` / `putPolygon` / `deletePolygon` を持つ
  - 各操作は即時にストレージへ反映し、`saveAll()` 一括保存は廃止（明示 flush 用途のみ残す）
- 他メンバーからの変更は `onRemoteChange(handler)` で受信し、エディタの `applyRemoteChange(ChangeSet)` で内部状態と UI に反映する
- 競合解決は LinkSelf の Last-Write-Wins（タイムスタンプ比較）に委ねる
  - 編集メンバーは通常異なる領域・区域を担当するため、同一エンティティへの同時操作は稀との前提
  - 辺の同時 split / delete のように整合が崩れた場合は orphan prune で復旧する
- Undo/Redo はローカル操作のみ対象とし、リモート由来の変更は対象外
- オフライン中の編集は LinkSelf の Store-and-Forward によりオンライン復帰時に自動伝播する

## ズーム制御

- 既定のズームステップ（1.0）は粗すぎるため、より細かい刻みでズームできるようにする
  - とくに最大ズーム付近とその一段下の差が大きすぎる
  - `zoomSnap` / `zoomDelta` を 0.25 程度に設定し、ホイール・ボタン双方で細かく制御できるようにする
- 最大ズームは 19 とする
  - GSI タイルのネイティブ最大は 18 だが、`maxZoom: 19` でわずかにオーバーズーム表示を許容する
  - それ以上の拡大は補間が粗くなり実用性が下がるため採用しない

## オフライン地図対応

- LinkSelfはオンライン時に必要な情報を取得する
- オンラインな場所で情報を取得してから現地に向かう運用とする

## 住宅情報

- 座標と紐づく「場所」モデルを起点とする
  - 場所は種別を持つ：戸建て、集合住宅、部屋
  - 集合住宅の場合は「部屋」である場所をリストとして保持する
  - オーナー情報は持たない（不動産管理ではないため）
  - 空室・入居中などの状態は訪問記録によって更新される
- マーカーを配置し住宅情報を管理する
  - 座標管理なので当家の意志により開示されている表札の情報だけ
    - 必要なら逆ジオコーディング機能を検討する
  - 集合住宅は部屋構成や部屋番号を記述できる方式を検討する
    - 部屋は各階ごとに横並びか
    - 大東建託のようなメゾネットタイプだと記述を工夫したほうが良いかもしれない
- これら住宅情報が活動メンバー向けアプリの訪問記録と紐づく

## 操作ヘルプ（ツールチップ）

- 地図画面を開いた直後、ポリゴン描画関連の操作ヘルプを自動で順次表示する
  - 対象はアプリ特有の操作のみ。ズーム/パン/レイヤ切替などの一般的な地図操作は対象外
  - 区域親番や区域の追加・削除もアイコンで直感的に分かるため対象外
- 実装済みのポリゴン描画操作のみをヘルプに含める。未実装の機能はヘルプに先出ししない
- 表示位置は画面右上、スライドダウンで出現し数秒後にフェードアウト
  - 同時表示は最大 5 件、超過時は最古から即時消去
  - 新しいメッセージは最上段に入り、既存は下へ押し下げる
- 各メッセージには「このメッセージを表示しない」リンクがあり、個別に永続非表示化できる
- 非表示状態は LinkSelf 個人設定に保存（端末間同期）
- 地図画面でのトリガ
  - データロード完了直後: 実装済みの描画操作 7 件を順次表示（セッション中 1 回）
  - ポリゴンの頂点ホバー: 「頂点をドラッグで移動」tip を再表示
  - ポリゴンの辺ホバー: 「辺を右クリックで新しい頂点を追加」tip を再表示
- **仕様・実装変更時は i18n の `tips.map.polygon.*` 文言と齟齬がないか必ず確認する**
- 実装済み対象操作（i18n キー `tips.map.polygon.*`）
  - `startDraw`, `continueVertex`, `confirmDraw`, `cancelDraw`
  - `selectPolygon`, `moveVertex`, `splitEdge`
